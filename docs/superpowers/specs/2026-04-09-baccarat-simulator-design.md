# 百家乐模拟器设计文档

## 概述

网页端百家乐模拟器，真实赌场风格，兼顾娱乐和统计分析功能。纯前端单文件实现，零依赖，支持移动端适配和 PWA。

## 技术方案

- **架构**：纯前端单文件（index.html），内联 CSS + JS
- **渲染**：DOM 布局 + Canvas 路单绘制
- **持久化**：localStorage
- **部署**：静态文件，零构建步骤

## 游戏核心逻辑

### 发牌规则

- 8 副牌（416 张），Fisher-Yates 洗牌
- 牌值：A=1, 2-9=面值, 10/J/Q/K=0，取个位数
- 剩余不足 6 张时重新洗牌

### 闲家补牌

- 合计 0-5：补牌
- 合计 6-7：不补
- 合计 8-9：天牌，双方不补

### 庄家补牌（闲家补牌后）

| 庄家合计 | 闲家第3张为以下时补牌 |
|---------|---------------------|
| 0-2     | 总是补              |
| 3       | 闲第3张不是 8       |
| 4       | 闲第3张 2-7         |
| 5       | 闲第3张 4-7         |
| 6       | 闲第3张 6-7         |
| 7       | 不补                |

若闲家不补牌，庄家 0-5 补，6-7 不补。

### 赔率

| 下注 | 赔率 | 抽水 |
|------|------|------|
| 庄赢 | 1:1  | 5%   |
| 闲赢 | 1:1  | 无   |
| 和   | 8:1  | 无   |
| 庄对 | 11:1 | 无   |
| 闲对 | 11:1 | 无   |

和局时庄/闲下注退回。

## 路单系统

### 珠盘路（Bead Plate）

- 6 行 N 列，从上往下从左到右填充
- 庄=红圆点，闲=蓝圆点，和=绿圆点
- 对子用边缘小标记（左下闲对，右上庄对）

### 大路（Big Road）

- 同一结果连续向下排列（同列）
- 结果改变换下一列
- 和局不换列，标绿线
- 6 行，超出拐弯龙尾

### 大眼仔（Big Eye Boy）

- 从大路第 2 列第 2 行开始推导（或第 3 列第 1 行）
- 齐整=红色空心圆，不齐整=蓝色空心圆

### 小路（Small Road）

- 从大路第 3 列第 2 行开始推导（或第 4 列第 1 行）
- 跳一列比较
- 齐整=红实心圆，不齐整=蓝实心圆

### 蟑螂路（Cockroach Pig）

- 从大路第 4 列第 2 行开始推导（或第 5 列第 1 行）
- 跳两列比较
- 齐整=红斜线，不齐整=蓝斜线

### 渲染

- Canvas 绘制，桌面端格子 20×20px，移动端 14×14px
- 自动滚动到最新数据
- 点击可展开全屏查看

## 统计面板

- 总局数、庄/闲/和 胜次与百分比
- 庄连胜最长、闲连胜最长、当前连胜
- 庄对/闲对次数
- 当前余额、盈亏、最大盈利、最大亏损

## 筹码系统

- 固定初始额度 10,000
- 筹码面额：10 / 50 / 100 / 500 / 1000
- 余额不足时无法下注
- 可重置余额回 10,000

## 多桌系统

- 最多 3 张桌子，顶栏 Tab 切换
- 每桌独立状态：牌局历史、路单、余额、统计
- 切换桌子时保存当前桌数据

## 持久化

### localStorage Key

```
baccarat_tables   → 桌子列表与状态
baccarat_table_1  → 桌子1完整数据
baccarat_table_2  → 桌子2完整数据
baccarat_table_3  → 桌子3完整数据
baccarat_settings → 用户设置
```

### 存储策略

- 每局结束后自动保存
- 切换桌子时保存
- beforeunload 兜底保存
- 每桌最多 500 局历史

## 交互设计

### 下注

- 先选筹码面额，再点下注区域
- 可叠加下注，右键/长按撤回
- "清除下注"一键清空
- "重复上局"复制上局下注

### 快捷键（桌面端）

- 1-5：选筹码面额
- P/T/B：下注闲/和/庄
- Space：发牌

### 结果展示

- 直接显示最终牌面，无动画
- 点数大字显示，胜方高亮
- 盈亏用 +绿/-红 闪现

## 重置功能

- 重置当前桌：清空当前桌数据，余额回 10,000
- 重置全部：清空所有数据
- 重置前弹出确认对话框

## PWA 支持

- manifest.json 内联到 HTML（data URI `<link>`）
- 独立 `sw.js` 文件（唯一的附属文件），缓存 index.html 实现离线
- 支持"添加到主屏幕"
- 最终交付物：`index.html` + `sw.js` 两个文件
